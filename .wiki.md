# Project Summary
TrackAjo is a digital infrastructure platform that modernizes traditional Ajo (rotating savings) systems into a secure, transparent, and automated fintech solution. It enables group administrators to create structured savings circles, manage contributions digitally, and ensure secure transactions through virtual bank accounts powered by Paystack. The platform enhances financial transparency, reduces fraud, and facilitates remote participation for underserved users.

# Project Module Description
The project consists of two main modules:
1. **Frontend**: A landing page built with Vite, utilizing plain JavaScript for improved compatibility and performance. The frontend allows users to join a waitlist for the TrackAjo service.
2. **Backend**: An Express.js server with a MongoDB database that manages waitlist entries and provides API endpoints for signup and count retrieval.

# File Description Inventory
- **index.html**: The main HTML file that serves as the entry point for the application, now configured to properly load scripts.
- **package.json**: Contains metadata and dependencies for both frontend and backend.
- **script.js**: Contains the application logic, implemented in plain JavaScript for broader compatibility.
- **style.css**: Custom CSS styles for the application.
- **template_config.json**: Configuration settings for templates.
- **todo.md**: Documentation of the development plan and tasks.
- **server/README.md**: Instructions on how to set up and run the backend server.
- **models/Waitlist.js**: Mongoose schema for the waitlist entries, including validation.
- **server/package.json**: Lists backend dependencies and scripts for running the server.
- **server/server.js**: Main file that sets up the Express server and defines API endpoints.

# Technology Stack
- **Frontend**: Vite, plain JavaScript, Bootstrap
- **Backend**: Express.js, Mongoose (MongoDB)
- **Database**: MongoDB (local or MongoDB Atlas)

# Usage
1. **Frontend**:
   - Navigate to the `frontend` directory.
   - Open `index.html` in a web browser to view the landing page.

2. **Backend**:
   - Navigate to the `server` directory.
   - Install dependencies: `npm install`
   - Set your MongoDB connection string: `export MONGO_URI="your_mongodb_uri"`
   - Start the server: `npm start`
